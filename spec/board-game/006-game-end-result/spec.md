# 기능 명세서: 게임 종료 및 결과 산정

**기능 브랜치**: `006-game-end-result`  
**작성일**: 2026-04-20  
**상태**: Draft  
**입력 설명**: "게임이 종료되는 조건을 판정하고, 승자와 최종 결과를 산정한다."

---

## 1. 기능 개요 *(필수)*

### 기능 설명

이 기능은 게임 종료 조건을 판정하고 최종 승자를 결정한다.

게임은 활성 플레이어가 1명만 남았거나, 최대 턴 수에 도달했거나, 제한 시간이 종료되었을 때 끝난다.

종료 시 각 플레이어의 최종 자산을 계산한다. 최종 자산은 보유 골드와 도시 평가액을 합산한다. 파산하지 않은 플레이어 중 최종 자산이 가장 높은 플레이어가 승리한다.

### 해결하려는 문제

게임 종료 조건이 명확하지 않으면 게임이 무한히 진행되거나, 승자 판정이 불공정하게 이루어질 수 있다.

### 기대 효과

- 게임 한 판의 명확한 완료 조건이 생긴다.
- 파산 승리와 시간 제한 승리를 모두 지원할 수 있다.
- 게임 결과 화면과 기록 저장의 기준이 생긴다.

### 완료 조건

- 활성 플레이어가 1명 남으면 게임이 종료된다.
- 최대 턴 수에 도달하면 게임이 종료된다.
- 종료 시 최종 자산을 계산한다.
- 최종 승자를 결정한다.
- 게임 상태가 `FINISHED`로 변경된다.
- 종료 이후 추가 액션이 차단된다.

## 2. 기능 범위 *(필수)*

### 포함 범위 (In Scope)

- 파산 기반 종료 조건
- 최대 턴 기반 종료 조건
- 최종 자산 계산
- 승자 결정
- 동점 처리
- 게임 결과 생성
- 종료 이벤트 로그 기록

### 제외 범위 (Out of Scope)

- 랭킹 점수 반영
- 경험치 지급
- 보상 상자 지급
- 시즌 기록
- 리플레이 저장
- 매치메이킹 결과 반영

### MVP 범위

본 기능의 MVP는 다음 종료 조건을 지원한다.

- 활성 플레이어가 1명만 남으면 즉시 종료
- 100턴에 도달하면 종료
- 종료 시 `money + propertyValue`가 가장 높은 플레이어가 승리
- 동점이면 보유 골드가 더 많은 플레이어가 승리
- 그래도 동점이면 공동 승리로 처리

## 3. 기능 시나리오 *(필수)*

### 시나리오 1 - 마지막 생존자 승리 (우선순위: P1)

다른 모든 플레이어가 파산하고 한 명만 활성 상태로 남으면 게임이 종료된다.

**행위자**: 시스템

**왜 중요한가**: 가장 기본적인 승리 조건이다.

**독립 테스트 방법**: 3명 게임에서 2명을 `BANKRUPT` 상태로 만들고 종료 판정 함수를 호출한다.

**인수 조건**:

1. **Given** 활성 플레이어가 1명만 남았을 때, **When** 종료 조건을 검사하면, **Then** 게임은 종료되어야 한다.
2. **Given** P1만 활성 상태일 때, **When** 게임이 종료되면, **Then** winnerId는 P1이어야 한다.
3. **Given** 게임이 종료되었을 때, **When** 추가 주사위 굴림 요청이 오면, **Then** `GAME_ALREADY_FINISHED` 에러가 반환되어야 한다.

---

### 시나리오 2 - 최대 턴 도달 종료 (우선순위: P1)

게임이 최대 턴 수에 도달하면 자산 기준으로 승자를 결정한다.

**행위자**: 시스템

**왜 중요한가**: 게임이 지나치게 길어지는 것을 방지한다.

**독립 테스트 방법**: `turnNumber`를 100으로 설정하고 종료 판정 함수를 호출한다.

**인수 조건**:

1. **Given** `turnNumber`가 100일 때, **When** 종료 조건을 검사하면, **Then** 게임은 종료되어야 한다.
2. **Given** P1의 최종 자산이 5,000,000이고 P2의 최종 자산이 4,000,000일 때, **When** 승자를 계산하면, **Then** winnerId는 P1이어야 한다.
3. **Given** 최종 자산이 동일할 때, **When** 보유 골드가 더 많은 플레이어가 있으면, **Then** 해당 플레이어가 승리해야 한다.

---

### 시나리오 3 - 최종 결과 생성 (우선순위: P2)

게임 종료 후 플레이어별 순위와 결과 정보를 생성한다.

**행위자**: 시스템

**왜 중요한가**: 결과 화면과 기록 저장에 필요하다.

**독립 테스트 방법**: 종료된 `GameSession`을 입력해 `GameResult`를 생성하고 순위, 자산, 승자 필드를 검증한다.

**인수 조건**:

1. **Given** 게임이 종료되었을 때, **When** 결과를 생성하면, **Then** 모든 플레이어의 최종 자산이 포함되어야 한다.
2. **Given** 게임이 종료되었을 때, **When** 결과를 생성하면, **Then** 플레이어 순위가 포함되어야 한다.
3. **Given** 공동 승리가 발생했을 때, **When** 결과를 생성하면, **Then** `winnerIds`에 여러 플레이어가 포함되어야 한다.

## 4. 예외 상황 및 경계 조건 *(필수)*

- 활성 플레이어가 0명인 경우 최종 자산 기준으로 승자를 결정한다.
- 최종 자산과 보유 골드가 모두 같으면 공동 승리로 처리한다.
- 파산 플레이어도 결과 목록에는 포함한다.
- 파산 플레이어의 최종 자산은 현재 money와 남은 propertyValue 기준으로 계산한다. 단, MVP에서는 파산 시 propertyValue는 0이다.
- 게임이 이미 종료된 상태에서 종료 판정을 다시 호출하면 기존 결과를 반환한다.
- 최대 턴 수는 기본 100턴으로 설정한다.
- 최대 턴 수가 1 미만이면 `MAX_TURN_INVALID`를 반환한다.

## 5. 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 활성 플레이어 수를 기준으로 종료 여부를 판단해야 한다.
- **FR-002**: 시스템은 활성 플레이어가 1명 이하이면 게임을 종료해야 한다.
- **FR-003**: 시스템은 `turnNumber`가 `maxTurn` 이상이면 게임을 종료해야 한다.
- **FR-004**: 시스템은 플레이어별 최종 자산을 계산해야 한다.
- **FR-005**: 시스템은 최종 자산이 가장 높은 플레이어를 승자로 결정해야 한다.
- **FR-006**: 시스템은 최종 자산 동점 시 보유 골드가 더 많은 플레이어를 승자로 결정해야 한다.
- **FR-007**: 시스템은 모든 기준이 동점이면 공동 승리로 처리해야 한다.
- **FR-008**: 시스템은 게임 종료 시 `GameSession` 상태를 `FINISHED`로 변경해야 한다.
- **FR-009**: 시스템은 게임 종료 후 모든 액션을 거부해야 한다.
- **FR-010**: 시스템은 게임 종료 이벤트와 결과를 로그로 기록해야 한다.

### 비기능 요구사항

- **NFR-001**: 종료 판정은 300ms 이내에 완료되어야 한다.
- **NFR-002**: 최종 결과 생성은 500ms 이내에 완료되어야 한다.
- **NFR-003**: 종료 판정은 동일 상태에 대해 항상 동일 결과를 반환해야 한다.
- **NFR-004**: `GameResult`는 JSON으로 직렬화 가능해야 한다.

## 6. 핵심 엔터티 *(데이터 관련 기능이면 필수)*

- **GameResult**: 게임 종료 결과를 의미한다. 주요 속성은 `sessionId`, `winnerIds`, `rankings`, `finishedReason`, `finishedAt`이다.
- **PlayerResult**: 플레이어별 결과를 의미한다. 주요 속성은 `playerId`, `rank`, `money`, `propertyValue`, `totalAsset`, `status`이다.
- **FinishedReason**: 종료 사유를 의미한다. 값은 `LAST_PLAYER_STANDING`, `MAX_TURN_REACHED`, `MANUAL_TERMINATION`이다.

### 최종 자산 계산 규칙

```text
totalAsset = money + propertyValue

propertyValue = sum(
  purchasePrice + sum(appliedUpgradeCosts)
)
```

파산 플레이어의 propertyValue는 MVP에서 0으로 처리한다.

- **SC-004**: 종료 후 액션 차단 테스트는 100% 통과해야 한다.

## 9. 가정사항 *(필수 권장)*

- MVP 최대 턴 수는 100턴으로 설정한다.
- MVP에서는 제한 시간을 적용하지 않는다.
- MVP에서는 수동 종료를 관리자 기능으로만 가정한다.
- 파산 플레이어는 게임 결과에는 포함되지만 승자가 될 수 없다.
- 공동 승리는 허용한다.

---

## 10. 오픈 이슈 / 확인 필요 사항

- 제한 시간 기반 종료 조건을 추가할지 결정이 필요하다.
- 공동 승리를 허용할지, 추가 타이브레이커를 둘지 결정이 필요하다.
- 파산 플레이어의 자산 평가 방식을 0으로 둘지, 파산 직전 기준으로 둘지 결정이 필요하다.
- 결과 저장소와 랭킹 반영 여부 결정이 필요하다.
