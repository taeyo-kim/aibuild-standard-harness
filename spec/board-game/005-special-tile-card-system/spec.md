# 기능 명세서: 특수 칸 및 카드 시스템

**기능 브랜치**: `005-special-tile-card-system`  
**작성일**: 2026-04-20  
**상태**: Draft  
**입력 설명**: "플레이어가 START, CHANCE, ISLAND, TRAVEL, TAX 등 특수 칸에 도착했을 때 발생하는 효과와 카드 시스템을 구현한다."

---

## 1. 기능 개요 *(필수)*

### 기능 설명

이 기능은 도시 칸이 아닌 특수 칸의 효과를 처리한다.

플레이어가 특정 특수 칸에 도착하면 해당 칸의 규칙에 따라 돈을 받거나 잃고, 특정 위치로 이동하거나, 일정 턴 동안 행동이 제한된다.

CHANCE 칸은 무작위 카드 효과를 발생시킨다. 카드는 이동, 보상, 벌금, 통행료 면제, START 이동 등의 효과를 가질 수 있다.

### 해결하려는 문제

특수 칸 효과가 명확하지 않으면 게임의 변동성, 전략성, 예외 처리가 부족해진다.

또한 카드 효과가 통행료, 이동, 파산, 턴 종료와 충돌할 수 있으므로 명시적 처리 순서가 필요하다.

### 기대 효과

- 게임의 재미와 변동성이 증가한다.
- 도시 구매 외의 다양한 이벤트가 발생한다.
- 특정 상태 이상, 이동 효과, 보상/벌금 효과를 테스트 가능한 방식으로 구현할 수 있다.

### 완료 조건

- START 칸 도착 효과를 처리할 수 있다.
- CHANCE 칸에서 카드 1장을 뽑아 효과를 적용할 수 있다.
- ISLAND 칸에 도착하면 플레이어를 고립 상태로 만든다.
- TRAVEL 칸에 도착하면 지정 가능한 도시로 이동할 수 있다.
- TAX 칸에 도착하면 세금을 지불한다.
- 특수 효과는 이벤트 로그에 기록된다.

## 2. 기능 범위 *(필수)*

### 포함 범위 (In Scope)

- START 칸 효과
- CHANCE 카드 뽑기
- 카드 효과 적용
- ISLAND 상태 적용
- ISLAND 탈출 조건
- TRAVEL 이동
- TAX 지불
- 특수 효과 이벤트 로그

### 제외 범위 (Out of Scope)

- 캐릭터 스킬
- 아이템 카드
- 유료 카드팩
- 카드 강화
- 시즌 이벤트 카드
- 애니메이션 연출
- 카드 일러스트

### MVP 범위

본 기능의 MVP는 다음 특수 칸 효과를 지원한다.

- START: 정확히 도착하면 300,000 골드 보너스 지급
- CHANCE: 미리 정의된 카드 중 1장을 랜덤 적용
- ISLAND: 2턴 동안 이동 불가
- TRAVEL: 원하는 CITY 칸 하나를 선택해 이동
- TAX: 300,000 골드 지불

카드 MVP 종류:

- 보너스 카드: 300,000 골드 획득
- 벌금 카드: 300,000 골드 지불
- START 이동 카드: START 칸으로 이동하고 START 보너스 획득
- 특정 칸 이동 카드: 5칸 앞으로 이동
- 통행료 면제 카드: 다음 통행료 1회 면제

## 3. 기능 시나리오 *(필수)*

### 시나리오 1 - CHANCE 카드 효과 적용 (우선순위: P1)

플레이어가 CHANCE 칸에 도착하면 카드 한 장을 뽑고 효과를 적용한다.

**행위자**: 시스템

**왜 중요한가**: 카드 효과는 특수 칸 시스템의 핵심이다.

**독립 테스트 방법**: 카드 결과를 고정한 테스트 랜덤 공급자를 사용하여 각 카드 효과를 검증한다.

**인수 조건**:

1. **Given** P1이 CHANCE 칸에 도착했을 때, **When** 카드 효과가 보너스 카드이면, **Then** P1은 300,000 골드를 받아야 한다.
2. **Given** P1이 CHANCE 칸에 도착했을 때, **When** 카드 효과가 벌금 카드이면, **Then** P1은 300,000 골드를 지불해야 한다.
3. **Given** P1이 CHANCE 칸에 도착했을 때, **When** 카드 효과가 START 이동 카드이면, **Then** P1의 위치는 0번 칸이 되어야 한다.

---

### 시나리오 2 - ISLAND 상태 처리 (우선순위: P1)

플레이어가 ISLAND 칸에 도착하면 일정 턴 동안 이동할 수 없다.

**행위자**: 시스템

**왜 중요한가**: ISLAND는 플레이어 행동을 제한하는 대표적인 상태 효과이다.

**독립 테스트 방법**: 플레이어를 ISLAND 칸에 배치하고 다음 턴에서 이동 가능 여부를 검증한다.

**인수 조건**:

1. **Given** P1이 ISLAND 칸에 도착했을 때, **When** 특수 칸 효과를 처리하면, **Then** P1의 `skipTurnCount`는 2가 되어야 한다.
2. **Given** P1의 `skipTurnCount`가 2일 때, **When** P1의 턴이 시작되면, **Then** P1은 주사위를 굴릴 수 없고 `skipTurnCount`는 1 감소해야 한다.
3. **Given** P1의 `skipTurnCount`가 0일 때, **When** P1의 턴이 시작되면, **Then** P1은 정상적으로 주사위를 굴릴 수 있어야 한다.

---

### 시나리오 3 - TRAVEL 이동 처리 (우선순위: P2)

플레이어가 TRAVEL 칸에 도착하면 이동 가능한 CITY 칸 중 하나를 선택해 이동한다.

**행위자**: 플레이어 / 시스템

**왜 중요한가**: 전략적 이동을 가능하게 한다.

**독립 테스트 방법**: TRAVEL 칸에 도착한 상태에서 특정 CITY `tileId`를 목적지로 선택하고 위치 변경을 검증한다.

**인수 조건**:

1. **Given** P1이 TRAVEL 칸에 도착했을 때, **When** P1이 CITY 칸을 선택하면, **Then** P1은 해당 CITY 칸으로 이동해야 한다.
2. **Given** P1이 TRAVEL 칸에 도착했을 때, **When** P1이 CITY가 아닌 칸을 선택하면, **Then** `TRAVEL_DESTINATION_INVALID` 에러가 반환되어야 한다.
3. **Given** P1이 TRAVEL로 상대 도시로 이동했을 때, **When** 이동이 완료되면, **Then** 해당 도시의 통행료 처리가 이어져야 한다.

## 4. 예외 상황 및 경계 조건 *(필수)*

- CHANCE 카드 덱이 비어 있으면 기본 카드 덱을 재생성한다.
- 카드 효과 적용 중 돈이 부족하면 파산 처리를 호출한다.
- ISLAND 상태의 플레이어는 주사위를 굴릴 수 없다.
- ISLAND 상태라도 더블 탈출 규칙은 MVP에서 지원하지 않는다.
- TRAVEL 목적지가 CITY가 아니면 이동을 거부한다.
- TRAVEL로 이동한 도착지가 상대 도시이면 통행료 처리를 이어서 실행한다.
- TAX 지불 시 돈이 부족하면 파산 처리한다.
- START 이동 카드로 START에 도착한 경우 START 도착 보너스를 지급한다.
- 특수 카드 이동으로 다시 CHANCE 칸에 도착해도 추가 카드는 뽑지 않는다. 무한 루프 방지를 위해 한 액션당 카드 효과는 1회만 적용한다.

## 5. 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 특수 칸 도착 시 tileType에 맞는 효과를 실행해야 한다.
- **FR-002**: 시스템은 START 칸 정확히 도착 시 300,000 골드를 지급해야 한다.
- **FR-003**: 시스템은 CHANCE 칸 도착 시 카드 1장을 선택해야 한다.
- **FR-004**: 시스템은 선택된 카드 효과를 플레이어 상태에 적용해야 한다.
- **FR-005**: 시스템은 ISLAND 칸 도착 시 플레이어의 `skipTurnCount`를 2로 설정해야 한다.
- **FR-006**: 시스템은 `skipTurnCount`가 1 이상인 플레이어의 주사위 굴림을 차단해야 한다.
- **FR-007**: 시스템은 TRAVEL 칸에서 CITY 목적지 선택 이동을 지원해야 한다.
- **FR-008**: 시스템은 TAX 칸 도착 시 300,000 골드를 차감해야 한다.
- **FR-009**: 시스템은 특수 효과로 돈이 부족해진 경우 파산 처리를 호출해야 한다.
- **FR-010**: 시스템은 모든 특수 효과 처리 결과를 이벤트 로그로 기록해야 한다.

### 비기능 요구사항

- **NFR-001**: 특수 칸 효과 처리는 500ms 이내에 완료되어야 한다.
- **NFR-002**: 카드 선택은 테스트 환경에서 고정 가능해야 한다.
- **NFR-003**: 카드 효과는 데이터 기반으로 추가/수정 가능해야 한다.
- **NFR-004**: 카드 효과 처리 중 무한 루프가 발생하면 안 된다.

## 6. 핵심 엔터티 *(데이터 관련 기능이면 필수)*

- **SpecialTileEffect**: 특수 칸 효과를 의미한다. 주요 속성은 `tileType`, `effectType`, `amount`, `targetRule`이다.
- **ChanceCard**: CHANCE 칸에서 뽑는 카드를 의미한다. 주요 속성은 `cardId`, `name`, `cardType`, `amount`, `moveDistance`, `description`이다.
- **PlayerStatusEffect**: 플레이어에게 적용된 상태 효과를 의미한다. 주요 속성은 `skipTurnCount`, `tollExemptionCount`이다.
- **SpecialEffectResult**: 특수 효과 처리 결과를 의미한다. 주요 속성은 `playerId`, `effectType`, `moneyDelta`, `positionChanged`, `nextRequiredAction`이다.

### 카드 타입

```text
BONUS_MONEY
PAY_FINE
MOVE_TO_START
MOVE_FORWARD
TOLL_EXEMPTION
```

### 특수 칸 타입

```text
START
CHANCE
ISLAND
TRAVEL
TAX
```

## 9. 가정사항 *(필수 권장)*

- MVP에서는 카드 덱 셔플을 단순 랜덤 선택으로 처리한다.
- MVP에서는 카드 보유 시스템을 지원하지 않는다. 단, 통행료 면제 카드는 상태 효과로 저장한다.
- MVP에서는 ISLAND 더블 탈출을 지원하지 않는다.
- MVP에서는 TRAVEL 이동 후 START 통과 보너스를 지급하지 않는다.
- 특수 이동으로 발생한 도착 칸 효과는 한 번만 처리한다.

## 10. 오픈 이슈 / 확인 필요 사항

- 카드 덱을 실제 덱처럼 소모형으로 운영할지 결정이 필요하다.
- ISLAND 탈출 조건에 더블 탈출을 추가할지 결정이 필요하다.
- TRAVEL에서 모든 CITY 이동을 허용할지, 특정 범위만 허용할지 결정이 필요하다.
- 통행료 면제 카드의 사용 시점을 자동으로 할지 선택형으로 할지 결정이 필요하다.
