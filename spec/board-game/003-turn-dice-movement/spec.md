# 기능 명세서: 턴, 주사위, 이동 시스템

**기능 브랜치**: `003-turn-dice-movement`  
**작성일**: 2026-04-20  
**상태**: Draft  
**입력 설명**: "플레이어가 자신의 턴에 주사위를 굴리고, 나온 값만큼 보드 위를 이동하며, 턴을 다음 플레이어에게 넘긴다."

---

## 1. 기능 개요 *(필수)*

### 기능 설명

이 기능은 게임의 턴 진행, 주사위 굴림, 플레이어 이동을 담당한다.

플레이어는 자신의 턴에만 주사위를 굴릴 수 있으며, 두 개의 6면 주사위 결과 합만큼 이동한다. 이동 중 START 칸을 통과하면 보너스 골드를 지급한다. 도착한 칸의 타입에 따라 이후 도시 구매, 통행료 지불, 특수 효과 처리가 이어진다.

### 해결하려는 문제

턴 순서와 이동 규칙이 명확하지 않으면 플레이어가 잘못된 시점에 액션을 수행하거나, 잘못된 위치로 이동하거나, START 보너스가 중복 지급될 수 있다.

### 기대 효과

- 게임 진행 순서가 명확해진다.
- 플레이어는 자신의 턴에만 액션할 수 있다.
- 주사위 결과와 이동 결과가 이벤트 로그로 남아 재현 가능하다.
- 이동 결과를 기반으로 후속 기능이 안정적으로 실행된다.

### 완료 조건

- 현재 턴 플레이어만 주사위를 굴릴 수 있다.
- 주사위는 1~6 사이 값 두 개를 생성한다.
- 플레이어는 두 주사위 합만큼 이동한다.
- START 통과 시 500,000 골드를 받는다.
- 이동 완료 후 도착 칸 정보가 반환된다.
- 턴 종료 시 다음 활성 플레이어에게 턴이 넘어간다.

## 2. 기능 범위 *(필수)*

### 포함 범위 (In Scope)

- 현재 턴 검증
- 주사위 굴림
- 더블 판정
- 플레이어 이동
- START 통과 보너스 지급
- 턴 종료
- 다음 플레이어 계산
- 이동 이벤트 기록

### 제외 범위 (Out of Scope)

- 도시 구매
- 통행료 지불
- 특수 칸 효과 실행
- 승패 판정
- 주사위 애니메이션
- 주사위 아이템, 스킬, 확률 조작

### MVP 범위

본 기능의 MVP는 현재 턴 플레이어가 두 개의 6면 주사위를 굴리고, 합산 결과만큼 이동한 뒤, 도착 칸 정보를 반환하는 것이다.

MVP 규칙:

- 주사위는 2개 사용한다.
- 각 주사위 값은 1~6이다.
- 이동 거리는 두 주사위의 합이다.
- START 통과 보너스는 500,000 골드이다.
- 더블이 나오면 한 번 더 턴을 진행할 수 있다.
- 연속 더블 3회 시 해당 플레이어는 ISLAND 칸으로 이동하고 턴이 종료된다.

## 3. 기능 시나리오 *(필수)*

### 시나리오 1 - 현재 턴 플레이어의 주사위 굴림 (우선순위: P1)

현재 턴인 플레이어가 주사위를 굴린다.

**행위자**: 플레이어

**왜 중요한가**: 턴 기반 게임의 기본 흐름이다.

**독립 테스트 방법**: 현재 턴 `playerId`로 `rollDice` 함수를 호출하고 `DiceResult`와 이동 결과를 검증한다.

**인수 조건**:

1. **Given** 현재 턴 플레이어가 P1일 때, **When** P1이 주사위를 굴리면, **Then** 주사위 결과와 이동 결과가 반환되어야 한다.
2. **Given** 현재 턴 플레이어가 P1일 때, **When** P2가 주사위를 굴리면, **Then** `NOT_PLAYER_TURN` 에러가 반환되어야 한다.
3. **Given** 게임 상태가 `WAITING`일 때, **When** 주사위를 굴리면, **Then** `GAME_NOT_IN_PROGRESS` 에러가 반환되어야 한다.

---

### 시나리오 2 - 플레이어 이동 (우선순위: P1)

주사위 합만큼 플레이어 위치를 이동시킨다.

**행위자**: 시스템

**왜 중요한가**: 이동은 게임 내 모든 이벤트 발생의 시작점이다.

**독립 테스트 방법**: 주사위 값을 고정한 테스트 랜덤 공급자를 사용하여 이동 결과를 검증한다.

**인수 조건**:

1. **Given** 플레이어 위치가 0이고 주사위 값이 3과 4일 때, **When** 이동하면, **Then** 플레이어 위치는 7이어야 한다.
2. **Given** 플레이어 위치가 30이고 주사위 값이 2와 3일 때, **When** 이동하면, **Then** 플레이어 위치는 3이어야 한다.
3. **Given** 플레이어 위치가 30이고 주사위 합이 5일 때, **When** START를 통과하면, **Then** 플레이어는 500,000 골드를 받아야 한다.

---

### 시나리오 3 - 더블 처리 (우선순위: P2)

두 주사위 값이 같으면 동일 플레이어가 한 번 더 턴을 가진다.

**행위자**: 시스템

**왜 중요한가**: 더블 규칙은 보드게임의 핵심 재미 요소이다.

**독립 테스트 방법**: 주사위 값을 동일하게 고정하고 턴이 유지되는지 검증한다.

**인수 조건**:

1. **Given** P1이 3과 3을 굴렸을 때, **When** 이동이 완료되면, **Then** 현재 턴은 P1으로 유지되어야 한다.
2. **Given** P1이 더블이 아닌 값을 굴렸을 때, **When** 턴이 종료되면, **Then** 현재 턴은 다음 플레이어로 변경되어야 한다.
3. **Given** P1이 연속 더블 3회를 기록했을 때, **When** 세 번째 이동을 처리하면, **Then** P1은 ISLAND 칸으로 이동하고 턴은 다음 플레이어에게 넘어가야 한다.

---

## 4. 예외 상황 및 경계 조건 *(필수)*

- 현재 턴이 아닌 플레이어의 주사위 요청은 `NOT_PLAYER_TURN`으로 거부한다.
- 게임 상태가 `IN_PROGRESS`가 아니면 주사위 요청을 거부한다.
- 파산 또는 탈락 상태의 플레이어는 주사위를 굴릴 수 없다.
- 주사위 값이 1~6 범위를 벗어나면 `DICE_RESULT_INVALID`를 반환한다.
- 이동 중 START를 여러 번 통과하는 경우 통과 횟수만큼 보너스를 지급한다. 단, MVP에서는 주사위 합 최대 12이므로 1회 이하만 발생한다.
- 더블 3회 발생 시 일반 도착 칸 효과를 실행하지 않고 즉시 ISLAND 이동 처리한다.
- 남은 활성 플레이어가 1명 이하인 경우 턴 진행을 중단하고 게임 종료 판정을 요청한다.

## 5. 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 현재 턴 플레이어만 주사위를 굴릴 수 있도록 검증해야 한다.
- **FR-002**: 시스템은 두 개의 6면 주사위를 굴려야 한다.
- **FR-003**: 시스템은 주사위 합을 이동 거리로 계산해야 한다.
- **FR-004**: 시스템은 이동 후 플레이어 위치를 갱신해야 한다.
- **FR-005**: 시스템은 START 통과 시 500,000 골드를 지급해야 한다.
- **FR-006**: 시스템은 주사위 값이 같은 경우 더블로 판정해야 한다.
- **FR-007**: 시스템은 더블이 아닌 경우 턴을 다음 활성 플레이어에게 넘겨야 한다.
- **FR-008**: 시스템은 연속 더블 횟수를 추적해야 한다.
- **FR-009**: 시스템은 연속 더블 3회 시 플레이어를 ISLAND 칸으로 이동시켜야 한다.
- **FR-010**: 시스템은 주사위 결과, 이동 전 위치, 이동 후 위치, START 보너스 지급 여부를 이벤트 로그에 기록해야 한다.

### 비기능 요구사항

- **NFR-001**: 주사위 굴림부터 이동 결과 반환까지 300ms 이내에 완료되어야 한다.
- **NFR-002**: 테스트 환경에서는 주사위 결과를 고정할 수 있어야 한다.
- **NFR-003**: 주사위 결과와 이동 결과는 재현 가능한 로그로 남아야 한다.
- **NFR-004**: 턴 변경은 원자적으로 처리되어야 한다.

## 6. 핵심 엔터티 *(데이터 관련 기능이면 필수)*

- **DiceResult**: 주사위 결과를 의미한다. 주요 속성은 `dice1`, `dice2`, `total`, `isDouble`이다.
- **TurnState**: 현재 턴 상태를 의미한다. 주요 속성은 `currentPlayerId`, `turnNumber`, `consecutiveDoubleCount`, `phase`이다.
- **MovementResult**: 이동 결과를 의미한다. 주요 속성은 `playerId`, `fromPosition`, `toPosition`, `distance`, `passedStart`, `startBonusAmount`, `landedTile`이다.
- **TurnPhase**: 턴의 세부 단계를 의미한다. 값은 `WAITING_FOR_DICE`, `MOVED`, `WAITING_FOR_ACTION`, `TURN_ENDED`이다.

## 8. 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 주사위 굴림 및 이동 처리는 300ms 이내에 완료되어야 한다.
- **SC-002**: 1,000회 테스트 주사위 결과 중 1~6 범위 밖 값 발생률은 0%여야 한다.
- **SC-003**: 이동 위치 계산 테스트는 100% 통과해야 한다.
- **SC-004**: 턴 권한 검증 테스트는 100% 통과해야 한다.
- **SC-005**: 더블 및 연속 더블 처리 테스트는 100% 통과해야 한다.

## 9. 가정사항 *(필수 권장)*

- MVP에서는 주사위 아이템이나 스킬을 사용하지 않는다.
- MVP에서는 주사위 확률은 균등 분포로 가정한다.
- MVP에서는 턴 제한 시간을 적용하지 않는다.
- 보드는 32칸으로 고정되어 있다고 가정한다.
- ISLAND 칸 index는 8이라고 가정한다.

## 10. 오픈 이슈 / 확인 필요 사항

- 주사위 연출 완료 전 상태 갱신 여부 결정이 필요하다.
- 더블 3회 시 벌칙 규칙을 변경할지 결정이 필요하다.
